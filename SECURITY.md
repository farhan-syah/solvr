# Security Policy

## Supported Versions

Security fixes are provided for the latest beta release line.

| Version      | Supported |
| ------------ | --------- |
| 0.0.x-beta   | Yes       |
| < 0.0.x-beta | No        |

## Reporting a Vulnerability

Please report vulnerabilities privately by opening a security advisory in the repository (GitHub Security Advisories).

When reporting, include:

- A clear description of the issue and impact.
- Reproduction steps or a minimal proof of concept.
- Affected versions and feature flags.
- Any suggested mitigation.

You can expect an initial response within 5 business days.

Do not open public issues for unpatched security vulnerabilities.
